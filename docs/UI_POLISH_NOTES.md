# UI Polish Notes

## Phase 8.1 - PortalShell and Navigation Polish

### Shell changes

- Portal shell responsibilities were split into smaller components:
  - `PortalShell`
  - `PortalSidebar`
  - `PortalTopbar`
  - `PortalMobileDrawer`
  - `PortalNavItem`
  - `portalNav`
- Verification banner behavior remains in `PortalShell` with the existing auth state and i18n copy.
- Topbar is intentionally lighter and no longer presents notification, message, or settings buttons as available actions.

### Navigation behavior

- Role navigation is now generated by `getPortalNavItems`.
- Active navigation is resolved by longest matching route through `getActivePortalNavHref`.
- `/app/profile` is exact-only so it does not collide with role routes.
- Parent dashboard routes are exact-only so child routes can win visually.
- Laboratory nav includes `/app/lab/scan`.
- Pharmacist nav includes `/app/pharmacist/scan` and `/app/pharmacist/history`.

### Visual rules established

- Sidebar uses a lighter role summary and tighter navigation density.
- Navigation icons use stable fixed-size containers.
- Active items use one primary treatment only.
- User identity text is constrained with wrapping/truncation support for long names and emails.
- Mobile drawer uses dialog semantics and opens from the right in Arabic/Kurdish, left in English.

### Remaining Phase 8.2 candidates

- Unify dashboard section structure across patient, doctor, laboratory, and pharmacist pages.
- Introduce shared `StatCard`, `WorkflowCard`, `Alert`, `LoadingState`, and `ErrorState` primitives.
- Standardize empty/loading/error layouts across role dashboards.
- Continue reducing large rounded-card nesting where pages feel too heavy.
